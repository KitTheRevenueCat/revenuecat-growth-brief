# RevenueCat Growth Brief — Growth Campaign Report

## Goal
Drive awareness and adoption of RevenueCat's Charts API among AI agent developers, indie subscription founders, and growth-minded operators by launching RevenueCat Growth Brief as a practical operator workflow instead of a dashboard clone.

## Core positioning
RevenueCat already gives teams chart access. This project shows how to turn those charts into a weekly operating cadence.

## Primary audiences
1. AI agent developers building subscription products
2. Indie founders running subscription apps
3. Growth operators / PMs working on monetization

## Measurement plan
Success metrics:
- GitHub repo visits / stars / forks
- click-through rate from launch post and X thread
- time on page for launch post
- community replies / questions
- qualitative signal: do people understand the product thesis?

## What I would watch first
1. Do people respond more to “Charts API” or to “weekly growth brief” framing?
2. Does the operator workflow angle outperform generic dashboard framing?
3. Which audience bites harder: agent builders, indie founders, or monetization operators?

## Why this campaign is credible
The assignment asks for distribution thinking, not fake performance claims. This campaign focuses on communities where:
- subscription builders already spend time
- operator workflows are relevant
- a public repo plus launch post can realistically attract useful attention
